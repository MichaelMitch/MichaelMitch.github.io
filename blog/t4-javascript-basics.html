How does JavaScript compare to HTML and CSS?

Javascript differs as it's an object oriented language. Instead of being able to set classes and values to things straight off the bat you have to declare each variable before you use it. Javascript is meant to create programs instead of creating a site. You don't build individual boxes and fill them with information and styles like you do with HTML or CSS, instead you build a series of instructions for the computer to follow and then compile it for use within a site. Javascript seems to deal with the collection of information through use of their created objects.

Explain the difference between accessing data from arrays and objects.

Accessing data from arrays is simple, in that all you really need to do is call the array with the location of the data you want. i.e dataIWant[4]. you can then use this in any operations you want to use, or store it into another variable. Sometimes the array is a nested array, and you may have to use a handful of locations to access the data you want. in that case you call dataIWant[4][5] to get to it. To access data from an object, you need to know the name of the variable within the object to retrieve it. sometimes the variable is only accessable to the object itself (written as this.dataIWant) and so you may need to call a function to get to it, these functions are usually called getter functions, and are written as get.DataIWant()

Explain what functions are and why they are useful.

Functions are extremely useful for code in terms of reusablility. if you have a process that you may need to redo many different times in your work, you can create it in a function and only have to call one line of code to use it. It doesn't have to be the same variables you pass to it too, as you can pass it variables, and the function will set it to the function variables you specified. once the varable has run, you can choose whether to return data that the function had ended at, or if you were just changing global variables with it you can just leave it at that.
